# 🚀 실행 방법

1. `npm i` 를 통해 node_modules 설치
2. 개발 서버 실행.
3. 브라우저에서 localhost:3000로 접속해서 결과 확인.

# 🛠️ 사용한 기술 스택 및 선택 이유

### 스타일링 도구

- tailwindCSS

별도의 스타일링 파일 없이 className으로 스타일링이 가능함.
네이밍에 고민 할 필요가 없음.

### 폼 & 유효성

- react-hook-form
- zod

게시글 작성 등에 필요한 폼 상태를 RHF으로 처리하며, Zod와 resolver 연동을 통해 유효성 검증해 유효성 검사는 Zod를 통해서만 할 수 있도록 관심사를 분리할 수 있다.
Zod 스키마로부터 타입을 자동으로 생성할 수 있어, 추후 백엔드와 구조를 통일해 사용하기 편리하다.

### 상태 관리

- zustand

가볍고 직관적이며 Recoil이나 Redux처럼 Provider로 감싸줄 필요가 없어 사용이 간편하다.
또한. persist 를 통해 로컬 스토리지 저장도 쉽게 처리가 가능하다.
게시글의 좋아요, 북마크, 리트윗등은 작은 단위로 상태가 나뉘는게 아닌, 전체 리스트를 하나로 관리해야 한다고 생각해서 jotai보단 zustand를 선택함.

### 무한 스크롤

- react-intersection-observer

IntersectionObserver보다 강단하게 사용할 수 있게 도와주며, 코드도 훨씬 깔끔해진다.
특히, 무한 스크롤 구현시 다음 데이터를 불러오는 패턴을 쉽게 구현할 수 있다.

# ✅ 구현한 기능 목록

### 1-1 게시물 리스트 페이지

- 무한 스크롤로 게시물 목록 표시

### 1-2 게시물 작성 페이지

- 게시물 작성 페이지로 구현
- 텍스트 입력 280자 제한 및 실시간 글자 수 카운터
- 이미지 첨부 기능 (미리보기 포함)
- 작성 완료 후 피드에 새 게시물 반영

### 2-1 좋아요 시스템

- 낙관적 업데이트 적용
- 클릭시 애니메이션 효과 적용
- 게시글 좋아요 Toggle (아이콘 상태 변화 , 카운트 증가/감소)

# 💡 추가 구현한 기능

- 게시글 리트윗 Toggle (아이콘 상태 변화 , 카운트 증가/감소) 낙관적 업데이트 적용
- 게시글 북마크 Toggle (아이콘 상태 변화 , 카운트 증가/감소) 낙관적 업데이트 적용
- 북마크 페이지 추가 (북마크한 게시물만 모아보기)

# 🤔 기술적 고민과 해결 과정

### 구조

api 가 따로 없이 더미데이터를 통해서 예시로 해야하는 부분이 처음엔 어떻게 구조를 잡아야하는걸까 고민을 많이 했지만,
api 스펙이 예시와 같이 정해진다고 생각하고 추후에 api 연결하기 좋게 작업을 해보자라는 생각으로 작업을 했습니다.

### 1. 상호 작용 기능 구현

각 게시글에 있는 리트윗, 좋아요, 북마크 등의 버튼을 눌렀을때, 아이콘이 표시 되면서 카운트가 증가/감소가 되어야하는 부분에 있어 단순 한번 동작하는게 아닌 새로고침후에도 변경된 데이터를 가지고 있게 하기 위해서 zustand 를 활용해 post의 데이터를 sessionStorage에 저장해 동기화를 시키도록 했다. 로컬스토리지에 저장되어있는 데이터를 프로젝트의 mockup 정적 데이터로 싱크를 맞출 순 없어서, 처음 스토리지에 값이 있는경우에는 페이지를 맞춰서 동일한 mockup데이터가 중복되지 않도록 했습니다.

### 2. 북마크 페이지 구현

게시글에 있는 북마크를 선택하는경우 북마크만 모여있는 페이지가 있는게 좋다는 생각이 들어,
처음에는 mockup 데이터에서 isBookmarked의 값을 필터링해서 사용하려고 했지만, 그렇게 사용하게 되면 실시간으로 북마크를 사용하는 게시글 북마크로 보여지는게 어려워서 스토리지에 저장된 값을 필터링해 같이 사용하게 하기 위한 구조 고민을 했습니다.
